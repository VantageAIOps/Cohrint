// cohrint index v0 · coming Q3 2026

The first public benchmark of
AI coding efficiency

Every engineering team running AI coding tools is flying blind on cost and quality. The Cohrint Index changes that — anonymized data from real teams, updated quarterly, published under a permissive license.

// what it measures

Four metrics that actually matter

We deliberately excluded vanity metrics (autocomplete acceptance rate, lines-of-code generated) and focused on what CFOs and platform engineers care about.

COST PER USEFUL OUTPUT
USD spent per accepted code block, merged commit, or resolved task. Normalized by team size and workload type.
ROUTING EFFICIENCY RATIO
What fraction of spend went to the cheapest model that could handle the task. A score of 1.0 = perfect routing.
QUALITY DRIFT RATE
How often a cheaper routed model produced output rated equivalent to the premium model. Sampled at 5% of traffic.
SPEND CONCENTRATION
What % of total AI spend comes from the top 5% of requests. High concentration = addressable waste.
// methodology

How we collect and protect the data

No prompts. No responses. No PII. Every data point is a number — tokens, cost, latency, quality score — derived from routing decisions the proxy made locally on the engineer's machine.

01

Participating teams install the local proxy

The Cohrint local proxy runs on-device. Prompts and responses never leave the machine. Only aggregated stats — token counts, cost, routing decision, quality delta — are sent.

02

Each team's data is anonymized before aggregation

Org IDs are hashed. Team names are replaced with random slugs. No company name, engineer name, or project name appears in the dataset.

03

Minimum cohort size of 50 teams before publication

We will not publish v0 until at least 50 teams are represented. This ensures no individual team's data is identifiable through inference.

04

Raw anonymized data published under CC0

The full dataset is released under the Creative Commons Zero license — no attribution required, no restrictions. Researchers, analysts, and practitioners can use it freely.

team_slugstringRandom anonymized ID (e.g. "team_7f3a")
team_size_bucketstring1–5, 6–15, 16–50, 50+
periodstringYYYY-MM (monthly rollup)
total_spend_usdfloatTotal AI coding tool spend that month
routed_spend_usdfloatSpend that went through Cohrint routing
savings_usdfloatDocumented savings from routing decisions
routing_efficiencyfloat0–1 score (1.0 = every call optimally routed)
quality_drift_ratefloat% of sampled calls where cheaper model matched premium
top_intentstringMost common intent: autocomplete / generation / refactor / explanation
primary_providerstringopenai / anthropic / google
