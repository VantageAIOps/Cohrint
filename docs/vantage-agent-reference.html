Boot Sequence
🔴
HARD GATE: Do not write code, read source files, or answer codebase questions until all three steps below complete successfully. Sub-agents: run this too — never inherit parent context.
1
Load Memory + CLAUDE.md
Locate memory path, read index, read project rules.
# Find memory path ls ~/.claude/projects/ 2>/dev/null | grep -i vantage # Read memory index (from matched path) Read: ~/.claude/projects/<matched>/MEMORY.md # Read project rules Read: <project_root>/CLAUDE.md # If path not found → log: "Memory not found — proceeding without it"
2
Orient to Current Work
Check branch, recent commits, open PRs.
git status --short git log --oneline -5 gh pr list --state open --json number,title,headRefName
3
Check CI Status
Always check CI before assuming anything is broken or working.
gh run list --limit 3 --json status,conclusion,name,headBranch # For detailed failure output: gh run view <run-id> --log-failed
⚠️
After boot: For every file/function name found in memory — verify it still exists with Glob/Grep before acting on it. Memory can be stale.
📖
Golden Rule — Read Before Write
✅
Always: Read the actual file with the Read tool → verify function signatures, field names, table names → then write code. If this doc contradicts the source file, trust the source file.
Before touching any file, run:
# Verify route exists grep -n "app\.\(get\|post\|patch\|delete\)" vantage-worker/src/routes/<file>.ts # Verify table schema grep -n "CREATE TABLE <tablename>" vantage-worker/src/ -r # Verify helper signature before writing tests grep -n "def fresh_account\|def signup_api\|def get_headers" tests/helpers/api.py # Check for recent migrations git log --oneline --all -- "vantage-worker/src/db*" "vantage-worker/migrations*" | head -10

Common Gotchas
🕐
Timestamp formats differ by table events.created_at = INTEGER unix epoch. cross_platform_usage timestamps = TEXT 'YYYY-MM-DD HH:MM:SS'. Never apply unix epoch assumptions to cross_platform queries.
🔢
D1 batch response format Returns {accepted: N, failed: M} — not a single status code. Always check both fields.
🏷️
Team alias in JOINs Always alias e.team when joining with team_budgets to avoid ambiguity. SELECT e.team AS event_team ...
🔑
SSE token is one-time-use 32-char hex, 120s TTL, generated at /v1/auth/session. Never re-use across page loads.
📦
OTel field path is deeply nested resourceMetrics[].scopeMetrics[].metrics[] — navigate the full path, not shortcuts.
🍪
Cookie domain must be root Set on vantageaiops.com (no subdomain) to share across api. and app.
🧪
fresh_account() returns a 3-tuple Always: api_key, org_id, cookies = fresh_account(prefix="xx") — all three, every time.
🛡️
Brute-force counter increments on failure only Not on every request. 10 failed attempts / 5 min → lock. Don't write tests that assume per-request increment.
🗄️
otel_events table may not exist on older deploys Created lazily. OTel inserts are wrapped in try/catch — a silent failure on old deploys is expected.
🔐
RBAC Gap — Viewer Guard
🔴
Known architectural risk: Viewer enforcement is an inline guard inside each handler — NOT enforced at middleware. Every new write route MUST manually add the check.
Required on every POST / PATCH / DELETE handler:
// TypeScript — add at top of every write handler if (auth.role === 'viewer') return c.json({ error: 'Forbidden' }, 403) // Verify it exists before creating PR: grep -n "role.*viewer\|viewer.*403\|Forbidden" vantage-worker/src/routes/<file>.ts
Missing this = viewer-role accounts silently get write access.
🚫
Never Do
✗Push to main directly — always branch → PR → CI → merge
✗Push to a merged branch — always create new branch from main
✗git push --force origin main
✗git reset --hard without explicit approval
✗git add -A or git add . — always add specific files
✗console.log any vnt_* token
✗Commit API keys, wrangler.toml IDs, or .env files
✗Skip auth on any route — even health/ping endpoints
✗Hard-delete user data — soft delete (deleted_at) only
✗Concatenate user input into SQL — parameterized queries only
✗wrangler delete or DROP TABLE on production
✗Re-use SSE tokens across page loads (one-time use, 120s TTL)
✗Mock the database in tests — all tests hit live APIs
✗Write code for roadmap items unless explicitly starting that sprint

Schema Rules (Quick Ref)
TableTimestamp TypeNote
events INTEGER unix epoch Never ISO 8601. Composite PK (id, org_id) → INSERT OR IGNORE = idempotent
cross_platform_usage TEXT 'YYYY-MM-DD HH:MM:SS' Exception to the rule. period_start/period_end are TEXT not integer
sessions INTEGER (expires_at) 64-char hex token, 30-day TTL
otel_events TEXT (timestamp) Created lazily — may not exist. Inserts wrapped in try/catch
orgs INTEGER (created_at) budget_usd=0 means no budget set (not zero budget)
